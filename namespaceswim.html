◆ cost()

double swim::cost ( long  t)
inline

Time "cost function" computes the cost of adding a record to a report, once the record has age t seconds.

It uses a quadratic function, scaled by a constant kTimeDecayConstant. We will continue to add records to our report, until the total "cost" of doing so exceeds a given budget, given by kTimeDecayBudget.

◆ operator<()

bool swim::operator< ( const ServerRecord &  lhs,
const ServerRecord &  rhs 
)

Ordering operator for server records, uses the hostname as a total ordering criterion. For servers on the same host (IP), the port number is used; the timestamp is not used as a sorting criterion.

The sorting by hostname is done solely lexicographically, no meaning is assigned to the either the name or the IP address (if present); it is simply a means to allow us to store ServerRecord objects into a set.

Parameters
lhsthe left-hand side ServerRecord for the comparison
rhsthe right-hand side of the comparison
Returns
whether lhs is less than rhs, according to a lexicographic ordering of their respective ServerRecord::server()
