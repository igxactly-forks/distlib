◆ onPingRequest()

void swim::SwimServer::onPingRequest ( Server *  sender,
Server *  destination 
)
protectedvirtual

Invoked when the client sends a SwimEnvelope::Type::STATUS_REQUEST message to this server.

This is essentially a callback method that will be invoked by the server's loop, in its own thread.

This message requests this server to send a ping on behalf of the sender, to the destination server; if the latter responds within the given timeout, this server will send back a report to notify that it is alive.

If the destination does not respond, no action is taken, beyond marking it as a suspected crashed server.

Note that this server class is currently NOT thread-safe and that none of its members (apart from the atomic<bool> stopped_) should be modified while processing this call.

Parameters
senderthe server that just sent the message; the callee obtains ownership of the pointer and is responsible for freeing the memory.
destinationthe other server that the requestor (client) is asksing this server to ping and report status back.
timestampwhen the message was sent (according to the client's clock).

◆ DEFAULT_POLLING_INTERVAL_MSEC

const unsigned long swim::SwimServer::DEFAULT_POLLING_INTERVAL_MSEC = 50
static

Default value for the interval in msec for the server to wait for incoming data, between checks of having been stopped.

This is a timeout value, so there is no adverse effect to set it at a higher value, however this will impact the time it takes for the server to stop or shutdown.
